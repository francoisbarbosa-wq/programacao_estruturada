O QUE É UMA STRUCT?
💡 ANALOGIA

Imagine uma ficha de livro em uma biblioteca física. Ela contém título, ano e preço juntos — não em fichas separadas. A struct faz exatamente isso em C.

Problema: guardar 3 dados separados é repetitivo e desorganizado:
char titulo[100] ↦ pertence ao livro
int ano ↦ pertence ao livro
float preco ↦ pertence ao livro
↓
Com struct, você cria um novo tipo que agrupa tudo. O operador . acessa cada campo: livro1.ano = 2020;

struct Livro { char titulo[100]; int ano; float preco; }; // Declarar e usar: struct Livro livro1; livro1.ano = 2020;

struct Livro { /* seus campos aqui */ }; struct Livro cadastrarLivro(void) { struct Livro l; /* leia os campos com scanf */ return l; } int main(void) { struct Livro livro = cadastrarLivro(); /* exiba os campos do livro */ return 0; }

O QUE É TYPEDEF?
PROBLEMA

Sem typedef, você escreve struct Livro em todo lugar — na declaração de variável, no retorno de função, nos parâmetros. Fica verboso e repetitivo.

SOLUÇÃO: TYPEDEF

typedef cria um apelido para um tipo existente. Depois, Livro funciona exatamente como int ou float — sem precisar escrever struct na frente.

✅ REGRA PRÁTICA
Sempre que usar uma struct mais de duas vezes, crie um typedef. Seu código fica mais limpo e fácil de ler.
SEM TYPEDEF
struct Livro { char titulo[100]; int ano; }; struct Livro l1; struct Livro l2; struct Livro ler(void); void ver(struct Livro l);
COM TYPEDEF ✓
typedef struct { char titulo[100]; int ano; } Livro; // apelido! Livro l1; Livro l2; Livro ler(void); void ver(Livro l);

typedef struct { char nome[80]; char nacionalidade[50]; } Autor; typedef struct { char titulo[100]; int ano; float preco; Autor autor; // struct dentro da struct } Livro; Autor cadastrarAutor(void) { Autor a; /* leia nome e nacionalidade */ return a; } Livro cadastrarLivro(void) { Livro l; /* leia titulo, ano, preco */ l.autor = cadastrarAutor(); return l; } // Acesso aninhado: // l.autor.nome
