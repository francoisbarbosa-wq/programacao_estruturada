SEÇÃO 02

Regras Gerais

RG-01

Tudo em C

O programa deve ser implementado integralmente em linguagem C.

RG-02

Proibido ponteiros

Nenhum *, aritmética de ponteiros ou malloc/calloc/free.

RG-03

Array de structs

Armazenamento em vetor com tamanho fixo: #define MAX 10.

RG-04

total como retorno

Funções que mudam a quantidade retornam o novo total (int), reatribuído na main.

RG-05

Modularização

Código organizado em funções, proibido concentrar a lógica na main.

RG-06

Struct aninhada + menu

Ao menos uma struct dentro de struct, e menu interativo em do-while.

SEÇÃO 03

Estruturas de Dados obrigatórias

estruturas.c
struct Cargo {
    char  titulo[30];
    float salarioBase;
};

struct Funcionario {
    int  codigo;
    char nome[50];
    struct Cargo cargo;  // aninhada
    int  anosEmpresa;
};

#define MAX 10
i
Os campos acima são o mínimo exigido. Não remova nenhum campo — você pode acrescentar outros, desde que a struct aninhada Cargo seja preservada.
!
Altere os modelos e use typedef em todo o algoritmo, criando os tipos Cargo e Funcionario sem repetir a palavra struct.
POR QUÊ?

A struct aninhada obriga você a acessar dados em dois níveis: func.cargo.salarioBase.

SEÇÃO 04

Funcionalidades exigidas

O menu deve conter, no mínimo, as seguintes opções:

1 Cadastrar funcionário
2 Editar funcionário
3 Deletar funcionário
4 Buscar funcionário
5 Listar todos os funcionários
0 Sair
i
O menu permanece ativo em um laço do-while e só encerra quando o usuário escolhe a opção 0 — Sair.
SEÇÃO 05

Assinaturas de Funções obrigatórias

prototipos.c
// Exibe o menu de opcoes na tela
void exibirMenu(void);

// Insere um novo funcionario ao final do array e retorna o novo total
int  cadastrarFuncionario(struct Funcionario empresa[], int total);

// Localiza pelo codigo e sobrescreve os dados (senao, "nao encontrado")
void editarFuncionario(struct Funcionario empresa[], int total, int codigo);

// Remove pelo codigo (deslocando elementos) e retorna o novo total
int  deletarFuncionario(struct Funcionario empresa[], int total, int codigo);

// Retorna o INDICE do funcionario no array, ou -1 se nao existir
int  buscarFuncionario(struct Funcionario empresa[], int total, int codigo);

// Lista todos, exibindo os dados do struct Cargo aninhado
void listarFuncionarios(struct Funcionario empresa[], int total);
!
editarFuncionario não retorna total: ela não altera a quantidade de elementos, apenas sobrescreve dados já existentes.

SEÇÃO 07

Esqueleto da main (obrigatório)

main.c
int main(void) {
    struct Funcionario empresa[MAX];
    int total = 0, opcao, codigo;
    do {
        exibirMenu();
        scanf("%d", &opcao);
        switch (opcao) {
            case 1: total = cadastrarFuncionario(empresa, total); break;
            case 2: editarFuncionario(empresa, total, codigo); break;
            case 3: total = deletarFuncionario(empresa, total, codigo); break;
            case 4: {
                int idx = buscarFuncionario(empresa, total, codigo);
                if (idx != -1) printf("Encontrado: %s\n", empresa[idx].nome);
                else printf("Funcionario nao encontrado.\n");
                break;
            }
            case 5: listarFuncionarios(empresa, total); break;
            case 0: printf("Encerrando...\n"); break;
            default: printf("Opcao invalida!\n");
        }
    } while (opcao != 0);
    return 0;
}

SEÇÃO 08

Cadastrar funcionário

Adiciona um novo funcionário na posição empresa[total] (logo após o último cadastrado) e devolve o novo total para a main reatribuir.

  1. 1Verificar espaço. Checar total < MAX. Se cheio, exibir erro e retornar o total inalterado, nunca escrever fora dos limites.
  2. 2Validar código duplicado. Chamar buscarFuncionario antes de inserir. Se retornar índice ≠ -1, o código já existe → recusar e retornar o total sem alteração.
  3. 3Inserir e incrementar. Passou nas validações? Gravar em empresa[total] e retornar total + 1.
SEÇÃO 09

Buscar funcionário

  1. 1Percorrer o array. Um for de 0 até total - 1 (nunca até MAX, só posições preenchidas são válidas).
  2. 2Comparar o código. Verificar empresa[i].codigo == codigo.
  3. 3Retornar ao encontrar. Devolver imediatamente o índice i (busca sequencial com saída antecipada).
  4. 4Sentinela -1. Se o laço terminar sem correspondência, retornar -1 = "não encontrado".
FUNÇÃO-BASE

Reutilizada por todas

Editar, deletar e as opções 4/6 dependem dela. Qualquer erro aqui se propaga, implemente com cuidado.

i
Retornar o índice (int) em vez do struct é o que elimina o ponteiro: o índice é uma "referência segura" para ler empresa[idx].nome ou modificar o elemento no array original.
SEÇÃO 10

Editar funcionário

Localiza um funcionário pelo codigo e atualiza seus dados, inclusive os campos do Cargo aninhado, sobrescrevendo a posição no array. Não retorna total: a quantidade não muda.

  1. 1Localizar. Reaproveitar buscarFuncionario para obter o índice (evita duplicar lógica).
  2. 2Tratar não encontrado. Se retornar -1, avisar que o código não existe e encerrar sem pedir dados.
  3. 3Exibir dados atuais. Boa prática: mostrar o registro antes de sobrescrever, para confirmar a pessoa certa.
  4. 4Coletar novos dados. Gravar em empresa[idx], incluindo cargo.titulo e cargo.salarioBase. O codigo é a chave, não deve ser editável.
  5. 5Confirmar. Exibir mensagem de sucesso ao final.
