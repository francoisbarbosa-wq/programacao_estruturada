02
Motivação
😰 O problema

Como se fazia
antes das structs

Arrays paralelos — um por campo
nomes[], matriculas[], notas[] — todos no mesmo índice i
O índice vira a "cola" entre os dados
aluno 0 → nomes[0], matriculas[0], notas[0]
Qualquer erro de índice = desastre
deletar o aluno 3 exige remover em todos os arrays
Passar para função = 4 parâmetros
função que imprime precisa dos 4 arrays + N
/* Dados de 40 alunos — jeito antigo */ #define N 40 char nomes[N][40]; int matriculas[N]; float notas[N]; int faltas[N]; /* Preencher aluno i */ scanf("%s", nomes[i]); scanf("%d", &matriculas[i]); scanf("%f", &notas[i]); scanf("%d", &faltas[i]); /* Imprimir aluno i */ printf("%d", matriculas[i]); printf("%s", nomes[i]); printf("%.1f", notas[i]); printf("%d", faltas[i]);
❌ Se i ficar errado, todos os campos ficam descasados.
E ainda temos 4 arrays para manter sincronizados.
03
Motivação
Antes vs Depois

Arrays paralelos → struct

❌ Antes — arrays paralelos
/* 4 arrays, 1 índice por aluno */ char nomes[40][40]; int matriculas[40]; float notas[40]; int faltas[40]; /* passar para função: */ void imprimir( char nomes[][40], int matriculas[], float notas[], int faltas[], int n );
✅ Depois — com struct
/* 1 tipo, tudo junto */ typedef struct { int matricula; char nome[40]; float nota; int faltas; } Aluno; Aluno turma[40]; /* passar para função: */ void imprimir( Aluno turma[], int n );
04
O que é struct?
🗂️ Analogia

Pense em
uma ficha
cadastral

Toda ficha reúne dados diferentes sobre um mesmo sujeito.
Uma struct faz exatamente isso em C.
Sem struct → múltiplos arrays paralelos
nomes[], matriculas[], notas[], faltas[]…
Com struct → 1 variável organizada
Aluno joao; — tudo junto!
Ficha de Aluno
Matrícula 2024001 int
Nome João Silva char[40]
Nota Final 8.5 float
Faltas 3 int
struct Aluno
= a ficha inteira em uma só variável
05
Sintaxe
Declaração

Como declarar
uma struct

struct — palavra-chave do C
indica que vem um tipo composto
Cada campo tem seu tipo e nome
separados por ; como variáveis normais
}; — ponto-e-vírgula obrigatório!
erro clássico: esquecer o ; após o }
A declaração não ocupa memória — apenas define o tipo.
Memória é alocada ao criar uma variável desse tipo.
struct Aluno { int matricula; char nome[40]; float nota; int aprovado; }; /* ← ponto-e-vírgula! */ /* declarando variáveis */ struct Aluno a1; struct Aluno a2;
struct Produto { int codigo; char descricao[60]; double preco; int estoque; }; struct Produto p;
06
Operador ponto
Acesso

Operador ponto .

📦 Analogia
Pense em a1 como uma caixa.
O ponto . é a gaveta que você abre
para acessar o campo desejado.
a1.matricula → lê/escreve inteiro
a1.nota → lê/escreve float
a1.nome → array de char
usar strcpy() para escrever, %s para imprimir
struct Aluno a1; /* Escrevendo nos campos */ a1.matricula = 2024001; a1.nota = 8.5f; a1.aprovado = 1; strcpy(a1.nome, "João Silva"); /* Lendo os campos */ printf("Matrícula: %d\n", a1.matricula); printf("Nome: %s\n", a1.nome); printf("Nota: %.1f\n", a1.nota); /* Lendo com scanf */ scanf("%d", &a1.matricula); scanf("%s", a1.nome); scanf("%f", &a1.nota);
07
typedef
Simplificando

typedef — apelido para o tipo

typedef cria um apelido para um tipo existente. Com structs, elimina a necessidade de escrever struct toda vez. É o padrão em código C profissional.
❌ Sem typedef — verboso
struct Aluno a1; struct Aluno a2; struct Aluno turma[40]; void imprimir(struct Aluno a) { /* ... */ }
✅ Com typedef — idiomático
typedef struct { int matricula; char nome[40]; float nota; } Aluno; Aluno a1; Aluno turma[40]; void imprimir(Aluno a) { /* ... */ }
08
Inicialização
Três formas

Inicializando
uma struct

Campo a campo
mais verboso, mais explícito
Com chaves — posicional
segue a ordem de declaração dos campos
Com chaves — por nome (C99)
mais legível, ordem livre
⚠️ Dica
Campos não inicializados contêm lixo.
Sempre inicialize todos antes de usar!
/* 1. Campo a campo */ Aluno a1; a1.matricula = 2024001; strcpy(a1.nome, "Maria"); a1.nota = 9.0f; /* 2. Chaves — posicional */ Aluno a2 = {2024002, "Pedro", 7.5f}; /* 3. Por nome de campo (C99) ✅ */ Aluno a3 = { .matricula = 2024003, .nome = "Ana", .nota = 8.0f }; /* Zerando todos os campos */ Aluno vazio = {0};
09
Cópia
Atribuição

Structs são copiadas por valor

a2 = a1; copia todos os campos de uma vez automaticamente. Modificar a2 não afeta a1.
✅ Cópia com =
Aluno a1 = {1, "Ana", 8.0f}; Aluno a2 = a1; /* cópia completa */ a2.nota = 9.5f; printf("a1: %.1f\n", a1.nota); /* Saída: a1: 8.0 ← não mudou */ printf("a2: %.1f\n", a2.nota); /* Saída: a2: 9.5 */
⚠️ Comparação com == NÃO funciona
/* ERRADO — erro de compilação: */ if (a1 == a2) { /* ← inválido */ } /* CERTO — compare campo a campo: */ if (a1.matricula == a2.matricula) { printf("Mesma matrícula\n"); } /* Para strings usar strcmp: */ if (strcmp(a1.nome, a2.nome) == 0) { printf("Mesmo nome\n"); }
10
Aninhamento
🪆 Analogia

Struct dentro
de struct

Como uma boneca russa que contém outra boneca, uma struct pode ter outra struct como campo.
Acesso encadeado com pontos:
a.endereco.cidade
Reutilize a struct interna
Endereco serve para Aluno, Funcionário, Loja…
typedef struct { char rua[60]; int numero; char cidade[40]; char cep[10]; } Endereco; typedef struct { int matricula; char nome[40]; Endereco endereco; float nota; } Aluno; /* Preenchendo campos aninhados */ Aluno a; a.matricula = 42; strcpy(a.endereco.cidade, "Caxias"); strcpy(a.endereco.cep, "65604-180"); a.endereco.numero = 100; printf("%s - %s\n", a.endereco.cidade, a.endereco.cep);
11
Vetores
📚 Analogia

Vetor de structs
= prateleira
de fichas

Gaveta com 40 fichas em sequência — é exatamente o que Aluno turma[40] representa na memória.
Cada posição = struct completa
turma[0], turma[1], ..., turma[39]
Acesso: turma[i].campo
índice + ponto + nome do campo
#define N 40 Aluno turma[N]; /* Lendo a turma inteira */ for (int i = 0; i < N; i++) { scanf("%d", &turma[i].matricula); scanf("%s", turma[i].nome); scanf("%f", &turma[i].nota); } /* Calculando média */ float soma = 0; for (int i = 0; i < N; i++) { soma += turma[i].nota; } float media = soma / N; printf("Média: %.2f\n", media); /* Buscando por matrícula */ int busca = 2024005; for (int i = 0; i < N; i++) { if (turma[i].matricula == busca) { printf("%s\n", turma[i].nome); break; } }
12
Aplicação Real
🏦 Sistema Bancário

Conta corrente
como struct

Cada conta tem dados fixos (agência, número, titular) e saldo variável. Struct modela isso perfeitamente.
Struct Conta agrupa tudo
Funções recebem a struct como parâmetro
Vetor de Conta = banco real
typedef struct { int numero; char agencia[8]; char titular[60]; double saldo; int ativa; } Conta; void depositar(Conta *c, double valor) { c->saldo += valor; printf("Depósito: R$ %.2f\n", valor); printf("Saldo: R$ %.2f\n", c->saldo); } void sacar(Conta *c, double valor) { if (valor > c->saldo) { printf("Saldo insuficiente!\n"); return; } c->saldo -= valor; printf("Saque: R$ %.2f\n", valor); } Conta c1 = {1001, "0001", "João", 500.0, 1}; depositar(&c1, 200.0); sacar(&c1, 100.0);
13
Memória
🧱 Por baixo do capô

Campos ficam
contíguos
na memória

Uma struct é um bloco contínuo de memória dividido em faixas — uma por campo, na ordem de declaração.
Aluno a1; printf("%p\n", &a1); printf("%p\n", &a1.matricula); /* base + 0 → mesmo endereço */ printf("%p\n", &a1.nome); /* base + 4 */ printf("%p\n", &a1.nota); /* base + 44 */ printf("%zu\n", sizeof(a1)); /* 48 bytes */
struct Aluno — 48 bytes
endereço base: 0x1000
+0
int matricula
4 bytes
2024001
+4
char nome[40]
40 bytes
"João\0…"
+44
float nota
4 bytes
8.5f
sizeof(struct Aluno)
= 4 + 40 + 4 = 48 bytes

O compilador pode adicionar padding.
Use sempre sizeof() para descobrir o tamanho real.
14
sizeof
Tamanho

Medindo uma
struct com
sizeof

sizeof retorna o tamanho em bytes. Com structs, é a única forma confiável — o compilador pode inserir padding para alinhamento.
sizeof(int) 4 bytes
sizeof(double) 8 bytes
sizeof(char[20]) 20 bytes
sizeof(Produto) ≥ 32 bytes*
* pode ser maior por causa do padding
typedef struct { int cod; double preco; char nome[20]; } Produto; Produto p; /* Tamanho do tipo */ printf("%zu\n", sizeof(Produto)); /* Tamanho da variável */ printf("%zu\n", sizeof(p)); /* Tamanho de um campo */ printf("%zu\n", sizeof(p.preco)); /* Tamanho de um vetor inteiro */ Produto estoque[100]; printf("%zu bytes\n", sizeof(estoque));
Regra de ouro: nunca use números fixos para tamanho.
Use sizeof() — o compilador calcula para você.
15
Boas Práticas
Dicas profissionais

Boas práticas com structs

📛
Nomes claros
Use PascalCase para o tipo:
Aluno, ContaBancaria

Use snake_case para campos:
nota_final, data_nascimento
🔄
Sempre typedef
Prefira:
typedef struct { ... } Nome;

Evita repetir struct Aluno em toda declaração. Padrão em C profissional e em bibliotecas como SDL e GTK.
🔢
Use sizeof()
Nunca assuma tamanhos fixos.

sizeof(Aluno) em vez de 48.

O padding pode variar entre compiladores e arquiteturas.
🧹
Inicialize sempre
Campos não inicializados = lixo.

Use = {0} para zerar tudo de uma vez.

Evita bugs silenciosos e difíceis de depurar.
📐
Agrupe campos
Campos do mesmo conceito → struct auxiliar.

Endereco dentro de Aluno
em vez de rua, numero, cidade soltos.
📝
Documente os campos
Comentários curtos nas declarações:

float nota; /* 0.0–10.0 */
int ativa; /* 1=sim 0=não */
