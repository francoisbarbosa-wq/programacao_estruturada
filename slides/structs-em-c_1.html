Motivação

Como se fazia antes das structs

  • Arrays paralelos — um por camponomes[], matriculas[], notas[] — todos no mesmo índice i.
  • O índice vira a "cola" entre os dadosaluno 0 → nomes[0], matriculas[0], notas[0].
  • Passar para função = 4 parâmetrose qualquer erro de índice descasa tudo.
/* 40 alunos — jeito antigo */ #define N 40 char nomes[N][40]; int matriculas[N]; float notas[N]; int faltas[N]; /* preencher o aluno i */ scanf("%s", nomes[i]); scanf("%d", &matriculas[i]); scanf("%f", ¬as[i]);

Antes vs. depois

Arrays paralelos → struct

Antes — arrays paralelos
/* 4 arrays, 1 índice */ char nomes[40][40]; int matriculas[40]; float notas[40]; int faltas[40]; void imprimir( char nomes[][40], int matriculas[], float notas[], int faltas[], int n);
Depois — com struct
/* 1 tipo, tudo junto */ typedef struct { int matricula; char nome[40]; float nota; int faltas; } Aluno; Aluno turma[40]; void imprimir(Aluno turma[], int n);
De 4 parâmetros sincronizados para 1 tipo que carrega tudo junto.

O que é struct?

Pense numa ficha cadastral

Toda ficha reúne dados diferentes sobre um mesmo sujeito. Uma struct faz exatamente isso em C.
  • Sem struct → vários arrays paralelosnomes[], matriculas[], notas[], faltas[]…
  • Com struct → 1 variável organizadaAluno joao; — tudo junto.

Sintaxe

Como declarar uma struct

  • struct — palavra-chave do Cindica que vem um tipo composto.
  • Cada campo tem tipo e nomeseparados por ; como variáveis.
  • }; — ponto-e-vírgula obrigatórioerro clássico: esquecer o ; após o }.
A declaração não ocupa memória — só define o tipo. A memória nasce ao criar uma variável.
struct Aluno { int matricula; char nome[40]; float nota; int aprovado; }; /* ← ponto-e-vírgula! */ struct Aluno a1, a2;
struct Produto { int codigo; char descricao[60]; double preco; int estoque; }; struct Produto p;

Acesso

Operador ponto .

Pense em a1 como uma caixa; o ponto . é a gaveta que abre o campo desejado.
  • a1.matricula → lê/escreve inteiro
  • a1.nota → lê/escreve float
  • a1.nome → array de charuse strcpy() para escrever, %s para imprimir.
struct Aluno a1; /* escrevendo nos campos */ a1.matricula = 2024001; a1.nota = 8.5f; strcpy(a1.nome, "João Silva"); /* lendo os campos */ printf("%d\n", a1.matricula); printf("%s\n", a1.nome); scanf("%f", &a1.nota);

Simplificando

typedef — apelido para o tipo

Cria um apelido para um tipo existente. Com structs, elimina escrever struct toda vez — é o padrão em C profissional.

Sem typedef — verboso
struct Aluno a1; struct Aluno a2; struct Aluno turma[40]; void imprimir(struct Aluno a) { /* ... */ }
Com typedef — idiomático
typedef struct { int matricula; char nome[40]; float nota; } Aluno; Aluno a1, turma[40]; void imprimir(Aluno a) { }

Inicialização

Inicializando uma struct

  • Campo a campomais verboso, mais explícito.
  • Com chaves — posicionalsegue a ordem de declaração dos campos.
  • Com chaves — por nome (C99)mais legível, ordem livre.
⚠ Dica: campos não inicializados contêm lixo. Use = {0} para zerar tudo.
/* 1. campo a campo */ Aluno a1; a1.matricula = 2024001; strcpy(a1.nome, "Maria"); /* 2. chaves — posicional */ Aluno a2 = {2024002, "Pedro", 7.5f}; /* 3. por nome de campo (C99) */ Aluno a3 = {.matricula=2024003, .nome="Ana"}; /* zerando todos os campos */ Aluno vazio = {0};

Atribuição

Copiadas por valor

a2 = a1; copia todos os campos de uma vez. Modificar a2 não afeta a1.

Cópia com =
Aluno a1 = {1, "Ana", 8.0f}; Aluno a2 = a1; /* cópia completa */ a2.nota = 9.5f; printf("%.1f\n", a1.nota); // 8.0 printf("%.1f\n", a2.nota); // 9.5
Comparação com == NÃO funciona
/* ERRADO — não compila: */ if (a1 == a2) { } /* compare campo a campo: */ if (a1.matricula == a2.matricula) { } /* strings → strcmp: */ if (strcmp(a1.nome, a2.nome) == 0) { }

Aninhamento

Struct dentro de struct

Como uma boneca russa que contém outra, uma struct pode ter outra struct como campo.

  • Acesso encadeado com pontosa.endereco.cidade
  • Reutilize a struct internaEndereco serve para Aluno, Funcionário, Loja…
typedef struct { char rua[60]; int numero; char cidade[40]; } Endereco; typedef struct { int matricula; char nome[40]; Endereco endereco; // aninhada } Aluno; Aluno a; strcpy(a.endereco.cidade, "Caxias"); a.endereco.numero = 100;

Vetores

Vetor de structs = prateleira de fichas

Uma gaveta com 40 fichas em sequência — é o que Aluno turma[40] representa na memória.

  • Cada posição = struct completaturma[0], turma[1], …, turma[39].
  • Acesso: turma[i].campoíndice + ponto + nome do campo.
#define N 40 Aluno turma[N]; /* ler a turma inteira */ for (int i = 0; i < N; i++) { scanf("%d", &turma[i].matricula); scanf("%s", turma[i].nome); scanf("%f", &turma[i].nota); } /* média da turma */ float soma = 0; for (int i = 0; i < N; i++) soma += turma[i].nota; printf("Média: %.2f\n", soma/N);

Aplicação real

Conta corrente como struct

Cada conta tem dados fixos (número, titular) e saldo variável. A struct modela isso com precisão.

  • A struct Conta agrupa tudo
  • Funções recebem a struct por ponteiroc->saldo altera o original (depositar, sacar).
  • Vetor de Conta = banco inteiro
typedef struct { int numero; char titular[60]; double saldo; int ativa; } Conta; void depositar(Conta *c, double v) { c->saldo += v; printf("Saldo: R$ %.2f\n", c->saldo); } Conta c1 = {1001, "João", 500.0, 1}; depositar(&c1, 200.0);

Memória

Campos ficam contíguos

Uma struct é um bloco contínuo de memória dividido em faixas — uma por campo, na ordem de declaração.

Aluno a1; printf("%p", &a1.matricula); // base+0 printf("%p", &a1.nome); // base+4 printf("%p", &a1.nota); // base+44 printf("%zu", sizeof(a1)); // 48
struct Aluno · base 0x1000 · 48 bytes
+0int matricula4 B2024001
+4char nome[40]40 B"João\0…"
+44float nota4 B8.5f
sizeof = 4 + 40 + 4 = 48 bytes. O compilador pode inserir padding — use sempre sizeof().

Tamanho

Medindo com sizeof

sizeof retorna o tamanho em bytes. Com structs é a única forma confiável: o compilador pode inserir padding.

sizeof(int)4 bytes
sizeof(double)8 bytes
sizeof(char[20])20 bytes
sizeof(Produto)≥ 32 bytes*
* pode ser maior por causa do padding
typedef struct { int cod; double preco; char nome[20]; } Produto; Produto p; printf("%zu\n", sizeof(Produto)); printf("%zu\n", sizeof(p.preco)); Produto estoque[100]; printf("%zu\n", sizeof(estoque));

Dicas profissionais

Boas práticas com structs

📛
Nomes claros
PascalCase no tipo (Aluno) e snake_case nos campos (nota_final).
🔄
Sempre typedef
typedef struct {…} Nome; evita repetir struct em toda declaração.
🔢
Use sizeof()
Nunca assuma tamanhos. sizeof(Aluno) em vez de 48 — o padding varia.
🧹
Inicialize sempre
Campos não inicializados = lixo. = {0} zera tudo de uma vez.
📐
Agrupe campos
Conceitos relacionados viram struct auxiliar: Endereco dentro de Aluno.
📝
Documente
Comentários curtos: float nota; /* 0.0–10.0 */.

Encerramento

Recapitulando

  1. Arrays paralelos são frágeis — a struct resolve.
  2. A struct agrupa dados heterogêneos num único tipo.
  3. Acesso aos campos via operador ponto (.).
  4. typedef elimina a repetição de struct.
  5. Structs são copiadas por valor com =.
  6. Vetor de structs = coleção de registros.
Da bagunça de arrays paralelos para dados organizados e reutilizáveis.
